State changes are modeled as Actions

                            }

The reducer

state + action = 'state


Reducer has to be a pure function!

The reducer


                            function reduce(state, action) {
                                switch (action.type) {
                                    case "ADD_TODO":
                                        return {
                                            ...state,
                                            todos: [
                                                ...state.todos,
                                                action.payload
                                            ]
                                        }
                                }

                                return state
                            }

What does this get us?

  • Easy debugging
  • Deterministic state changes
  • State and action history
  • "Time travel"

Cons

  • Lots of boilerplate
  • Most benefits require entire application to follow the pattern
  • The "linux problem"
  • Requires rethinking application architecture
  • Not viable for everything (e.g. forms)

Summary


  • Deterministic state changes
  • Easier debugging and inspection
  • Caching and minimizing API calls
  • Better performance and responsiveness
